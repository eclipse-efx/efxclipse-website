JavaFX Tooling and Runtime for Eclipse and OSGi

Incubation

e(fx)clipse is an incubating project.

Make sure you understand what that means by reading.

Prerequisites from external parties

You'll need at least JDK 7u7 installed in your system. Make sure your e(fx)clipse installation uses this JDK.

Download & Install we get you started

For the lazy

For the lazy ones who prefer an all-in-one download, we provide preconfigured packages so that you don't have to worry about all the different bits and pieces.

For the ambitious

For those who'd like to assemble their own e(fx)clipse installation based on Eclipse 4.3 or 3.8. we offer an installation guide which helps you get started.

Preconfigured distributions

If you don't want to use update sites but prefer an all-in-one download those downloads are provided by an external website http://efxclipse.bestsolution.at because this distro packages also other plugins not available directly from Eclipse.org (e.g. subclipse).

Update sites

To install the e(fx)clipse tooling into your already existing Eclipse you have to add an Update site. There's no released yet available so you need to add the nightly update-site http://download.eclipse.org/efxclipse/updates-nightly/site

If you are not familiar with update-sites you can follow the short guide below or use a pre-packaged version

Eclipse 4.3 & 3.8

1.

Download the latest release of the Eclipse 4.3 SDK or of the Eclipse 3.8 SDK from download.eclipse.org

2.

Fire up your Eclipse IDE if you have not done so already.

3.

e(fx)clipse is an extension for your Eclipse IDE which is delivered as a so called p2 repository. Extensions like this can be installed using the "Install New Software" wizard.

4.

Eclipse does not know about the location of the e(fx)clipse p2 repository so you need to add the repository as a software site. The address is: http://download.eclipse.org/efxclipse/updates-nightly/site

For Eclipse 3.8/Juno user
You have to add an additional update site for Xtext which is not preconfigured yet for your. The update site is http://download.eclipse.org/modeling/tmf/xtext/updates/composite/releases/

5.

In the tree of installable features, check only the entry corresponding to your release:

  • 4.3: e(fx)clipse - IDE - Kepler
  • 3.8: e(fx)clipse - IDE - Juno/Indigo

Also, make sure the "Contact all update sites during install to find required software" is checked.

You may, however, install the FX Runtime feature into your IDE but we strongly recommend composing your own target platform.

6.

Go through the wizard and let p2 do its job.

7.

When requested, restart Eclipse.
